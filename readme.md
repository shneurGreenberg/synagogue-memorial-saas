## Общая информация

 - Вам понадобится отдельный компьютер, который должен работать непрерывно круглые сутки, и который будет являеться веб-сервером в локальной сети Общинного центра.
   Веб-сервер будет хостить сайт на заданной в конфигурационном файле порте. Доступ по IP адресу и порту веб-сервера должен быть открыт для телевизора и планшета по локальной сути WiFi. По вопросам настройки локальной сети следует проконсультироваться с системным администратором Общины.
 - Если компьютер веб-сервера перезагрузился, или был отключен, веб-сервер следует перезапустить вручную, используя команду `yarn restart`.
 - Веб-сервер можно принудительно отключить командой `yarn stop`, после чего его можно запустить заново с помощью `yarn restart`.
 - Файл-конфигурации: [database.json](database.json). Рекомендую регулярно синхронизировать его содержимое на сервере с данным репозиторием. Так мне проще будет устранить возможные неполадки. Репозиторий приватный. Все данные в нём доступны только мне и Общине.
 - База данных лиц полностью представлена в том же файле конфигурации. Этот файл обслуживаете Вы(см. инструкции ниже).
 - Сайт автоматически перезагружается два раза в сутки на каждом устройстве, на котором он открыт.


## Как установить(с нуля)

1. Установите Node.js версии не ниже `v10.x`: https://nodejs.org/en/.
2. Установите pm2: https://www.npmjs.com/package/pm2.
3. Установите yarn: https://yarnpkg.com/.
4. Войдите в Github аккаунт Общины, и склонируйте данный репозиторий. Ваш аккаунт на Github: https://github.com/novosibirsk-synagogue. У Общины есть логин и пароль к аккаунту. Если аккаунт утерян, обратитесь ко мне, я могу его восстановить. Войти в аккаунт в Windows, и склонировать репозиторий можно с помощью Github Desktop: https://desktop.github.com/.
5. (Опционально) Настройте порт, на котором будет хоститься веб-сервер в файле конфигурации: [database.json](database.json#L2). По умолчанию установлен порт `3000`.
6. Введите команду `yarn restart`, чтобы перезагрузить сервер. Если сервер успешно перезагрузился, вы увидите надпись `Started at port 3000`.
7. Проверьте на телевизоре и планшете, что сайт работает.


## Как обслуживать список лиц

1. Записи всех людей находятся в файле конфигурации [database.json](database.json) (раздел `"people"`).
2. Вы можете добавлять и удалять записи.
3. Каждая запись состоит из следующих полей:
  - `id`. Должно быть любое уникальное целое положительное число, идентифицирующее запись. Без кавычек.
  - `name`. Имя и фамилия человека. Русскими буквами. То что будет отображаться на карточке. В двойных кавычках.
  - `gregorianDateOfDeath`. Дата смерти по грегорианскому календарю: месяц(`month`), день(`date`), год(`year`). Числа без кавычек. Первый месяц(Январь) `1`. Последний(Декабрь) `12`.
  - `photo`. Имя файла с фотографией человека. В двойных кавычках. Сам файл необходимо сохранить в директорию `photos`. Не перепутайте имена. Все имена файлов должны быть уникальными.
  - `title`. Краткий текст, который будет отображаться на карточке в день Кадиша. Текст в двойных кавычках. Это опциональное поле, его можно не заполнять.
  - `text`. Основной текст в формате html, который будет отображаться на планшете при просмотре карточки. В двойных кавычках. Внутри текста можно делать ссылки на других людей, используя код вида `<a href=\"/card/14\">Имя родственника</a>`, где число 14 соответствует идентификатору `id` другого человека.

  Пример записи:
  ```json
  {
    "id": 2,
    "name": "Name A11112",
    "gregorianDateOfDeath": {"month": 1, "date": 6, "year": 1950},
    "photo": "bohr.png",
    "title": "Текст",
    "text": "Подробный текст"
  }
  ```

  **Обратите внимание**, что крайне важно соблюдать корректность формата json, расставлять кавычки, запятые и скобочки. Иначе веб-сервер не заработает. Корректность можно проверить, например, [на этом сайте](https://jsonformatter.curiousconcept.com/), скопировав файл конфигурации целиком, и нажав кнопку Process.

4. После окончания редактирования **закоммитьте все изменения(включая фотографии) и запуште их в репозиторий на Github**.
5. Введите команду `yarn restart`, чтобы перезагрузить сервер. Если сервер успешно перезагрузился, вы увидите надпись `Started at port 3000`. Если что-то пошло не так, скорее-всего Вы допустили ошибку в форматировании `database.json`.
6. Перезагрузите страницу на телевизоре и на планшете, и убедитесь, что страница отображается корректно на обоих устройствах.


## Как добавить цитату дня

1. Все цитаты так же находятся в файле конфигурации [database.json](database.json) (раздел `"dailyCites"`).
2. Вы можете добавлять и удалять записи для каждого дня еврейского календаря.
3. Цитаты опциональны. Если за конкретный день нет цитаты, она просто не будет отображаться. Поэтому этот раздел можно заполнять постепенно. Не обязательно заполнять все дни.
4. Цитаты идентифицируются по еврейскому календарю по месяцам. Где первый месяц `1` соответствует месяцу Нисан, месяц `12` соответствует месяц Адар, месяц `13` соответствует месяцу Адар Бет.
5. Каждая запись состоит из следующих полей:
  - `"hebrewDate"`. В ней объект с полям месяца `month` и полям дня месяца `date`.
  - `text`. Текст цитаты в формате html. В двойных кавычках.

  Пример записи:
  ```json
  {
    "hebrewDate": {"month": 1, "date": 11},
    "text": "<b>Цитата для 11-го Нисана</b><br/>Текст цитаты"
  }
  ```

  **Обратите внимание**, что крайне важно соблюдать корректность формата json, расставлять кавычки, запятые и скобочки. Иначе веб-сервер не заработает. Корректность можно проверить, например, [на этом сайте](https://jsonformatter.curiousconcept.com/), скопировав файл конфигурации целиком, и нажав кнопку Process.

6. После окончания редактирования **закоммитьте все изменения(включая фотографии) и запуште их в репозиторий на Github**.
7. Введите команду `yarn restart`, чтобы перезагрузить сервер. Если сервер успешно перезагрузился, вы увидите надпись `Started at port 3000`. Если что-то пошло не так, скорее-всего Вы допустили ошибку в форматировании `database.json`.
8. Перезагрузите страницу на телевизоре и на планшете, и убедитесь, что страница отображается корректно на обоих устройствах.
